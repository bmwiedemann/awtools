Authentication

Some tools require a special user authentication. That is more than the per alliance authentication for most of the tools.

GE11 update: it is now possible to set a password for a user and log into the tools with that.
The primary method for user-authentication is to click an AWTools link within brownie.
The other method is to use AW forum authentication - but that is now disabled by default. you can enable it in the preferences2 tool
NOTE: for security reasons, this does only work while your IP address stays the same. After a modem reconnect your ISP might assign you a different IP. Then you need to log into AW again with brownie enabled.

General

How does Authentication work with brownie/AWTools? How does it know which alliance players belong to?
Brownie uses alliance membership from AW's alliances.csv file. There is a way to have additional untagged members (useful before PL1 or after reaching 200 points). This is set by those users themselves with the joinalli tool and the secret alliance password. When you click AWTools links from the brownie, a cookie is set in your browser which is then used to authenticate you (because brownie has seen you successfully sign in to AW with it).
You also get a similar cookie from the authawforum tool (now disabled by default, but you can enable it in preferences2 tool).
